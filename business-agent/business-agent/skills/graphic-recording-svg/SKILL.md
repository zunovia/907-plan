# グラレコ風SVGイラスト生成スキル

グラフィックレコーディング（グラレコ）のスタイルで、SVG/HTMLベースのイラストを生成する。外部API不要。Claude単体でコードベースの手描き風ビジュアルを作成する。

## グラレコとは

グラフィックレコーディングは、議論や概念をリアルタイムで視覚化する手法。特徴は：
- **手描き風**の線とアイコン
- **シンプルな人物**（棒人間〜簡略化した人物）
- **吹き出し**や**矢印**による関係性の表現
- **色は3-5色**に限定（マーカーペンの制約を模倣）
- **テキストと絵の融合**（文字も手書き風）

## SVGでの実現方法

### 手描き風の線の実装

SVGのpath要素に微細な揺らぎを加えることで手描き感を出す：

```javascript
// 手描き風の線を生成するヘルパー関数
function sketchyLine(x1, y1, x2, y2) {
  const midX = (x1 + x2) / 2 + (Math.random() - 0.5) * 4;
  const midY = (y1 + y2) / 2 + (Math.random() - 0.5) * 4;
  return `M${x1},${y1} Q${midX},${midY} ${x2},${y2}`;
}

// 手描き風の四角形
function sketchyRect(x, y, w, h) {
  const offset = () => (Math.random() - 0.5) * 3;
  return `M${x+offset()},${y+offset()} 
          L${x+w+offset()},${y+offset()} 
          L${x+w+offset()},${y+h+offset()} 
          L${x+offset()},${y+h+offset()} Z`;
}

// 手描き風の円（楕円で揺らぎを出す）
function sketchyCircle(cx, cy, r) {
  const points = [];
  for (let i = 0; i <= 360; i += 15) {
    const angle = (i * Math.PI) / 180;
    const wobble = r + (Math.random() - 0.5) * 3;
    points.push(`${cx + wobble * Math.cos(angle)},${cy + wobble * Math.sin(angle)}`);
  }
  return `M${points[0]} ${points.slice(1).map(p => `L${p}`).join(' ')} Z`;
}
```

### 基本アイコンライブラリ

以下のSVGパーツを組み合わせてグラレコを構築する：

#### 人物（簡略化）
- **棒人間**: 丸い頭 + 線の体。感情は顔の表情で表現
- **上半身人物**: 丸い頭 + 台形の体。より親しみやすい
- **グループ**: 3人の簡略化人物の集合

#### 吹き出し・バブル
- **丸い吹き出し**: アイデア・発言用
- **雲型バブル**: 思考・夢用
- **角丸四角形**: データ・ファクト用

#### 矢印・接続線
- **手描き矢印**: 方向性・フロー
- **波線矢印**: 影響・関連
- **双方向矢印**: 相互関係

#### コンセプトアイコン
- **電球**: アイデア・ひらめき
- **歯車**: プロセス・仕組み
- **ハート**: 共感・大切なこと
- **星**: 重要・ハイライト
- **旗**: ゴール・マイルストーン
- **時計**: 時間・スケジュール
- **グラフ**: データ・成長
- **本**: 知識・学び
- **握手**: 協力・パートナーシップ

## 出力モード

### モード 1: 単体イラスト（SVGファイル）
概念や議論を1枚の図解にまとめる。スライドに貼り付けて使用。

出力先: `/mnt/user-data/outputs/graphic-recording.svg` または `.html`

### モード 2: スライド埋め込み用
HTMLプレゼンテーションスキルやPPTXスキルと連携し、各スライドにグラレコ風イラストを挿入。

PptxGenJS連携の場合:
1. SVGをPNGに変換（puppeteer/sharpなど）
2. `slide.addImage()` でスライドに追加

HTMLプレゼン連携の場合:
1. SVGをそのままHTMLに `<svg>` タグで埋め込み

### モード 3: ビジュアルノート
議論やテキスト全体を1枚の大きなキャンバスにまとめる。A3/A2サイズ相当のSVG出力。

## ワークフロー

### Step 1: コンテンツ分析
ユーザーの入力（テキスト、スライド内容、議題など）から：
- **キーメッセージ**を3-7個抽出
- **関係性**（因果、並列、対比、時系列）を特定
- **感情トーン**（ポジティブ/中立/課題提起）を判定

### Step 2: レイアウト選択

| パターン | 用途 | 構成 |
|---|---|---|
| **センター放射型** | 1つの中心テーマ+周辺要素 | 中央に大きなアイコン+テーマ、放射状に関連要素 |
| **フロー型（横）** | プロセス・手順・時系列 | 左→右にステップが流れる |
| **フロー型（縦）** | タイムライン・変遷 | 上→下に時間が流れる |
| **比較型** | Before/After、2つの対比 | 左右に対比する要素を配置 |
| **マトリックス型** | 2軸での分類 | 4象限に要素を配置 |
| **ストーリー型** | 物語・経緯 | 曲線パスに沿ってシーンが並ぶ |
| **クラスター型** | カテゴリ分類 | 色分けされたグループの集合 |

### Step 3: SVG生成
- 手描き風関数を使ってSVG要素を配置
- テキストは手書き風フォント（Google Fonts: Caveat, Klee One, Yomogi など）
- 背景に薄い方眼やドット模様を追加（ノート感の演出）

### Step 4: 品質チェック
- テキストが重なっていないか
- 要素間のスペースが十分か
- 色のコントラストが適切か
- SVGがブラウザで正しく表示されるか

## デザインルール

### DO（やるべき）
- 線の太さを2-4pxに統一（手描き感）
- 影や立体感は**つけない**（フラットに）
- アイコンは**丸みを帯びた**シンプルな形
- テキストは短く（5-10文字/ラベル）
- 余白を多めに（要素間に十分なスペース）
- 手描き風の揺らぎを**すべての線**に適用

### DON'T（やってはいけない）
- 写真やリアルなイラストの使用
- グラデーション（フラットな塗りのみ）
- 影やドロップシャドウ
- 細すぎる線（1px以下）
- テキストの詰め込み（読めないものは意味がない）
- 左右完全対称のレイアウト（少しの非対称が手描き感）

## 日本語フォント推奨

Google Fontsから利用可能な手書き風日本語フォント：
- **Yomogi** — 丸みのある手書き風
- **Klee One** — 教科書体風で読みやすい手書き
- **Zen Kurenaido** — カジュアルな手書き
- **Yuji Syuku** — 筆ペン風

CSS埋め込み:
```css
@import url('https://fonts.googleapis.com/css2?family=Yomogi&family=Klee+One&display=swap');
```
