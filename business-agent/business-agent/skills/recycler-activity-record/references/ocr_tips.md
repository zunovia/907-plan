# 手書きOCR読み取りのコツ

## 基本原則

手書きPDFの読み取りは、このスキルで最もエラーが発生しやすい工程。
急がず、1ページずつ丁寧に確認すること。

## PDFの画像化

```python
from pdf2image import convert_from_path
pages = convert_from_path(filepath, dpi=200)
# 各ページをPNG保存してviewツールで確認
for i, page in enumerate(pages):
    page.save(f'img_{i}.png')
```

dpi=200が読み取りと処理速度のバランスが良い。

## 事業所PDFの読み取り

### 読み取る項目
1. **氏名**（ページ上部）— テンプレートの名前と完全一致させる
2. **事業所名**（丸で囲まれた箇所）— ただし金額との照合で最終判定する（教訓2参照）
3. **各行の日付**（左側の列）
4. **各行の金額**（右側の列）— 時間帯カテゴリ判定の根拠になる最重要項目

### よくある読み間違い
- 「1」と「7」の区別（手書きで酷似する場合がある）
- 「6」と「0」の区別
- 「3」と「8」の区別
- 日付の10の位が省略されている場合（例：「5」と書いてあるが実際は「15」）

### 事業所名の判定
丸囲みの文字だけでなく、**金額と諸謝金レート表**を照合すること。
- 金額が850/1,350/1,850 → Re☆ショップ系（あつた/さくら/よしの/物流補助）
- 金額が1,200/1,850/2,600 → 物流仕分け
- 金額が事業所名と矛盾する → 金額を信頼

## ステーションPDFの読み取り

### 特に注意すべき点
ステーションPDFは大量ページ（50ページ以上になることも）。流し読みは厳禁。

1. **日付**：「日付：20XX年 X月 X日」形式。年・月・日すべて正確に読む
2. **署名欄**：何名分の署名があるか正確に数える
   - 1名のみ → ステ1名セクション（1,700円）
   - 2名 → ステ2名セクション（1,200円）
3. **判読不能な漢字の署名** → **近藤昌宏さん**（Row 10）として処理

### ステーション署名のコツ
- 署名は崩し字が多い。名前リストと照合して最も近い人を特定する
- 完全に読めない漢字 → 近藤さん
- 名前が読めても確信が持てない場合は候補を挙げてユーザーに確認

## よしのPDFの読み取り

よしのセクションの手書きは他セクションより読みづらい傾向がある（202601検証で判明）。
- 1ページごとに日付を特に注意深く確認する
- 可能であれば、その月のカレンダーと照合して曜日の妥当性を確認する
- 不確実な日付はメモしておき、最後にまとめてユーザーに確認する

## 処理順序の推奨

1. まず全PDFを画像化する
2. 事業所PDF（あつた/さくら/よしの/物流）を先に処理（構造が統一的で読みやすい）
3. ステーションPDFは最後に処理（ページ数が多く集中力が必要）
4. 全PDF処理完了後、テンプレートの全員について「PDFに出現したか」確認する

## エラー防止チェックリスト

処理完了後、以下を確認：
- [ ] 全PDFのすべてのページを処理したか
- [ ] 同一人物が複数PDFに分散していないか確認したか
- [ ] 金額→時間帯カテゴリの変換は金額ベースで行ったか（時間ベースではなく）
- [ ] ステーションの署名人数（1名/2名）は正確か
- [ ] Row番号は今月のテンプレートから直接読み取ったか
