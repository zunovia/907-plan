# リサイクラー活動記録 自動記入スキル
**最終突合検証: 2026年2月データ ― 全65名・全項目一致確認済み**

## 概要
手書きの活動報告書PDFを画像として読み取り、Excelテンプレート（リサイクラー参加表）の正しいセルに日付を自動記入する。

## 過去の失敗から学んだ重要教訓（202601検証結果）

以下は202601データの答え合わせで発覚した系統的なエラー。毎月の作業で必ず意識すること。

### 教訓1: Row番号は毎月テンプレートから直接読み取る（キャッシュ厳禁）
テンプレートは月によって行の追加・削除がある（「NEW」マーク付きの行など）。
前月のRow-名前対応をそのまま使うと、さくら・よしのセクション全体がずれる大事故になる。
**毎月必ずテンプレートのシートを開き、全行の名前を読み取って最新のRow-名前マッピングを構築すること。**

### 教訓2: 事業所名は「丸囲み」ではなく「金額」で判定する
PDFの丸囲みは読み間違えやすい。事業所の判定は、金額と諸謝金レート表を照合して行うのが最も確実。
- 850/1,350/1,850円 → Re☆ショップ系（あつた/さくら/よしの/物流補助）
- 1,200/1,850/2,600円 → 物流仕分け
- 1,350/2,350/3,350円 → サブマネ（伊藤ひとみのさくらのみ）
金額が事業所判定と矛盾する場合は、金額を信頼して事業所を再確認すること。

### 教訓3: 時間帯カテゴリは金額から逆算する（時間の記載を信用しない）
PDFに書かれた作業時間は曖昧な場合がある。金額は正確に書かれているため、
**金額 → 諸謝金レート表 → 時間帯カテゴリ** の順で逆引きするのが最も正確。

### 教訓4: ステーションPDFは1ページずつ丁寧に読む
ステーションは55ページにも及ぶため流し読みしがちだが、日付・署名の読み取りミスが多発した。
- 日付欄の「年 月 日」を正確に読み取る
- 署名が1名か2名かを正確に判定する（1名=1,700円、2名=1,200円の固定レート）
- 判読不能な漢字署名 → 近藤昌宏さん（Row 10）として処理

### 教訓5: 全PDFのデータを漏れなく処理する
複数PDFに同一人物のデータが分散していることがある（例：伊藤ひとみ＝さくら＋物流仕分け）。
全PDFを処理し終えてから突合検証に入ること。1つのPDFを飛ばすと大量欠落になる。

### 教訓6: よしのセクションのPDF読み取りに特に注意
よしのの手書きは他セクションより読みづらい傾向がある。日付の読み間違いが系統的に発生した。
1ページごとに日付を声に出すつもりで確認すること。

## 処理フロー

### Step 1: PDFの画像化
```bash
pip install pdf2image Pillow openpyxl --break-system-packages
apt-get install -y poppler-utils
```
```python
from pdf2image import convert_from_path
pages = convert_from_path(filepath, dpi=200)
```

### Step 2: 画像の目視読み取り
**必ず先に `references/ocr_tips.md` を読んでから作業を開始すること。**

各ページの画像を`view`ツールで確認し、以下を読み取る：

**事業所PDF（あつた/さくら/よしの/物流）の場合：**
- 氏名（ページ上部）
- 事業所名（丸で囲まれた箇所）
- 各行の日付・金額

**ステーションPDFの場合：**
- 日付（「日付：2026年 X月 X日」）— 年月日すべて正確に読み取ること
- 出席確認サイン（署名欄、1〜2名）
- 署名が判読不能な漢字の場合 → **近藤昌宏さん**（Row 10）として処理
- 署名人数の判定：1名のみ→ステ1名セクション（1,700円）、2名→ステ2名セクション（1,200円）

### Step 3: テンプレート構造の理解
**必ず `references/template_structure.md` を確認すること。**

### Step 4: 金額→カテゴリ→列の判定
**必ず `references/amount_mapping.md` の判定ルールに従うこと。**

特に重要な判定:
- ステーションは**参加人数のみで日給が決まる**（1名=1,700円 / 2名=1,200円）。時間帯の分類は不要
- 物流の仕分け/補助は金額で自動判定可能
- 伊藤ひとみのさくらはサブマネ料金

### Step 5: 名前マッチング＆書き込み
**Row番号は前月の値を流用せず、必ず今月のテンプレートシートから直接読み取ること（教訓1参照）。**
テンプレートのRow番号と名前の対応を取り、正しいセルに日付を記入。
```python
# 毎月必須：テンプレートから最新のRow-名前マッピングを構築
name_row_map = {}
for row in range(5, ws.max_row + 1):
    name = ws.cell(row=row, column=2).value  # B列=氏名
    if name and str(name).strip():
        name_row_map[str(name).strip()] = row
```

### Step 6: 突合検証
書き込み後、以下を必ず検証:
1. 日付入力数 = 回数計
2. 支払額 = 回数 × 単価
3. 支給計 = 全セクション合算
4. 源泉所得税 = INT(支給計 × 10.21%)
5. 振込額 = 支給計 - 控除計
6. **エクセル関数の日給額 vs OCRで読み取った日給額に相違がないか確認。相違があれば即ユーザーに報告**（7〜9月は別単価の可能性あり等、料金体系が変わっている場合の検知用）
7. **関数の破損検知（後述の「関数破損チェック」セクション参照）**

## 重要ルール

### E〜L列（col 5〜12）
旧Verのため**一切使用しない**。日付記入はM列以降。

### 読めない文字の処理
読めない文字は「N/A」と記載し、ユーザーに確認する。

### テンプレートにない名前（重要：必ずユーザーにテーブルで報告）
手書きPDFにテンプレート（エクセル）に存在しない名前が出た場合、
**自己判断で無視・記入せず、全員分の詳細をテーブル形式でユーザーに報告すること。**

パターンの判断はユーザーが行う（Claude側で判断しない）。
福祉事務所の方は後日謝金支払が発生するため、日付・場所の記録が特に重要。

**報告テーブルのフォーマット（必須）：**

| # | 氏名 | 出現した事業所/場所 | 出現日 | 日給（読取値） | 一緒に活動した相方 | 備考 |
|---|---|---|---|---|---|---|
| 1 | 鈴木太郎 | Re☆創庫さくら | 2/5, 2/12, 2/19 | 1,350円 | — | 事業所PDFに3日分記載 |
| 2 | 山本花子 | ステーション（港） | 2/7 | — | 赤松好子 | 署名欄に2名記載 |
| 3 | 田中一郎 | 物流センター | 2/3, 2/10 | 1,850円 | — | 仕分け作業 |

テーブルの後に以下を添える：
```
上記の方はテンプレートに名前がありませんでした。
それぞれどのパターンに該当するか教えてください：
1. 新規入会の有償作業員（→テンプレートに行を追加して記入）
2. 新規入会の無償作業員（→記入不要）
3. 社員（→固定給のため記入不要）
4. 福祉事務所の方（→後日謝金支払あり。上記の日付・場所リストを保管）
```

**ユーザーの回答後の対応：**
- ①有償作業員 → テンプレートの「追加用」行 or 空き行に記入
- ②無償作業員 → 記入不要。作業完了
- ③社員 → 記入不要。作業完了
- ④福祉事務所の方 → エクセルへの記入は不要。ステーションで相方がいた場合、相方を2名セクションに記入。報告テーブルの日付・場所リストはユーザーが謝金処理に使用するため保管される

### 伊藤ひとみさん（サブマネ）
**さくらセクションのみ**サブマネ料金体系で計算。
あつた・物流など他セクションは通常料金。

### 複数セクション参加者
同一人物が複数事業所で活動する場合がある（例：伊藤ひとみ＝さくら＋物流仕分け）。
同じRowに複数セクションのデータを入れ、支給計は全セクション合算。

## 関数破損チェック（毎月必須）

エクセルの操作ミスで関数が壊れることがある。
同じ列の関数は（伊藤ひとみのさくら以外）全行で同一構造のため、
**上下の行と比較して異常値がある行は関数が壊れている可能性が高い。**

### チェック方法
テンプレートを`data_only=False`（数式モード）で読み込み、
集計列の数式を行間で比較する。

```python
import openpyxl
wb = openpyxl.load_workbook('template.xlsx')  # data_only=False（デフォルト）
ws = wb['202603']

# チェック対象の集計列（数式が入っている列）
check_cols = {
    'Y': 25,    # ステ1名支払額
    'AQ': 43,   # ステ2名支払額
    'BZ': 78,   # 物流仕分支払額
    'DC': 107,  # 物流補助支払額
    'ES': 149,  # あつた支払額
    'GE': 187,  # さくら支払額
    'HH': 216,  # よしの支払額
    'IA': 235,  # 支給計
    'IB': 236,  # 源泉所得税
    'ID': 238,  # 控除計
    'IE': 239,  # 振込額
}

# 各列について、全行の数式を取得し、行番号部分を除いた構造を比較
# 構造が異なる行 = 関数破損の可能性 → ユーザーに報告
```

### 伊藤ひとみの例外
さくら支払額（GE列, col 187）のみ、伊藤ひとみの行は他の行と数式が異なる（サブマネ料金参照）。
これは正常なので破損ではない。

### 報告フォーマット
```
【関数の破損の可能性を検知しました】
Row 34（伊藤和見）の ES列（あつた支払額）:
  この行の数式: =EP34*'諸謝金額と所得税率(10月～)'!E5+...（異常）
  他の行の数式: =EP34*'諸謝金額と所得税率(10月～)'!E17+...（正常）
  → 参照先が E5（ステーション料金）になっており、本来の E17（Re☆ショップ料金）と異なります。
  確認をお願いします。
```
